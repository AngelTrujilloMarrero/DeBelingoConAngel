# Plan: Canal de Telegram para Verbenas de Tenerife

## Objetivo
Publicar automáticamente información sobre eventos en un canal de Telegram, sin pedir cookies a los usuarios.

---

## Configuración Actual del Proyecto

- **Web**: https://debelingoconangel.web.app
- **Stack**: React + Vite + Firebase + Vercel
- **Backend**: Vercel Serverless Functions + Firebase Realtime Database
- **Detección de cambios**: Ya implementada en `recentActivity` (add/edit/delete/reagregado)

---

## Pasos para Implementar

### 1. Crear el bot en Telegram
- Abrir @BotFather
- /newbot
- Elegir nombre (ej: DeBelingoConAngel Bot)
- Obtener el token

### 2. Crear el canal
- Telegram → Nueva imagen (canal)
- Nombre: "Verbenas Tenerife" (o similar)
- Añadir bot como administrador:
  - Settings del canal → Administradores → Añadir bot
  - Dar permiso "Post messages"

### 3. Obtener Chat ID del canal
- El link del canal será: t.me/[nombre]
- El Chat ID suele ser: @[nombre] o -100xxxxxxxxxx
- Para obtenerlo: añadir el bot @userinfobot al canal y usar /getgroupid

### 4. Configurar Vercel
- Project Settings → Environment Variables
- Añadir TELEGRAM_BOT_TOKEN
- Añadir TELEGRAM_CHAT_ID

### 5. Crear las API functions
- Implementar src/utils/telegram.js
- Implementar api/telegram-send-weekly.js
- Implementar api/telegram-send-reminder.js
- Implementar api/telegram-notify-change.js

### 6. Configurar Crons en vercel.json
```json
{
  "crons": [
    {
      "path": "/api/telegram-send-weekly",
      "schedule": "0 20 * * 0"
    },
    {
      "path": "/api/telegram-send-reminder",
      "schedule": "0 20 * * 4"
    }
  ]
}
```

## Coste Estimado

- **Vercel**: Plan gratuito (suficiente)
  - 1,000,000 invocaciones/mes
  - ~60-100 invocaciones/mes para este uso
- **Firebase**: Plan gratuito (suficiente)
  - 100GB almacenamiento, 10K reads/mes para la web
- **Telegram**: Completamente gratis

---

## Notas Importantes

1. **No es cookie**: El canal de Telegram no usa cookies, los usuarios se unen voluntariamente
2. **GDPR**: No se recogen datos personales, solo el nombre de usuario de Telegram
3. **Los suscriptores controlan**: Los usuarios pueden abandonar el canal en cualquier momento
4. **Solo eventos de la semana**: No publicar histórico, solo la semana actual/siguiente
5. **Cancelaciones siempre inmediatas**: Sistema de alertas para cancelaciones
